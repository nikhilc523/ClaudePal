## Architecture Overview

```
Claude Code → ~/.claudepal/hook-forward.sh → curl POST
    → macOS HookServer (127.0.0.1:52429)
    → HookProcessor (actor, AsyncStream for decisions)
    → SQLite (GRDB)
    → [CloudKit push — DISABLED, code ready]
    → [iPhone CloudKit subscription — STUB, code ready]
```

**Hard rule:** macOS is always the authority. CloudKit is a projection. iPhone writes commands, Mac validates and resolves locally.

### Key Architecture Details

**HookProcessor** (`Sources/ClaudePalMacCore/HookServer/HookProcessor.swift`):
- Actor with `process()` for incoming hooks and `resolve()` for decisions
- Uses `AsyncStream<HookDecisionResponse>` — continuation stored synchronously on actor before suspension (fixes race condition from original CheckedContinuation approach)
- Optional `pusher: (any CloudSyncPushing)?` — defaults to nil, so tests work unchanged
- After each DB write, calls `await pusher?.pushSession/pushEvent/pushPendingDecision`
- Permission tools: Bash, Write, Edit, NotebookEdit, WebFetch, WebSearch
- Silent tools (auto-approved): Read, Glob, Grep, Agent, Skill, Task*, Plan*, AskUserQuestion
- Destructive detection: rm, git reset --hard, git push --force, drop table, truncate, dd

**Notch Panel** (`ClaudePalMac/NotchPanelController.swift`):
- `NSPanel` with `styleMask: [.borderless, .nonactivatingPanel]`, level `.statusBar`
- Compact (180x36) / Expanded (340xAuto) modes
- Auto-shows when pendingDecisions > 0 or active sessions, hides when idle
- Click-outside-to-collapse, screen change repositioning
- `KeyablePanel` subclass overrides `canBecomeKey` for SwiftUI button clicks

**CloudKit** (code written, disabled):
- `CloudKitManager.createIfAvailable()` checks entitlements + iCloud account before creating CKContainer
- Currently crashes without entitlements, so `startCloudKit()` is commented out in AppState
- Container: `iCloud.com.claudepal.sync`, Zone: `ClaudePalZone`
- Record types: CPSession, CPEvent, CPPendingDecision, CPApprovalCommand
- 30-second polling fallback for missed push notifications

### Design System

**Theme:** Dark mode, copper/orange accent (`#D4944B`), deep backgrounds
- `Color.cpBackground` — near-black `(0.04, 0.04, 0.05)`
- `Color.cpCard` — dark gray `(0.10, 0.10, 0.12)`
- `Color.cpAccent` — warm copper `(0.83, 0.58, 0.42)`
- `Color.cpApprove` — green, `Color.cpDeny` — red, `Color.cpWarning` — amber
- `CPGradient.hero` — copper-to-purple diagonal
- `CPFont` — system fonts with rounded design for headings, monospace for code
- `CPRadius.card = 16`, `CPRadius.hero = 24`
- Custom button styles: `CPPrimaryButtonStyle`, `CPSecondaryButtonStyle`
- `PulsingDot` — animated status indicator
- `CPHaptics` — light/medium/success/error feedback

**Terminal Mascot (`>_<`):**
- `TerminalMascot` view — rounded dark rect with traffic light dots (centered), `>_<` face in copper, random blinking animation
- Used as header icon in Dashboard and app icon
- App icon generated via `scripts/generate-icon.swift` — CoreGraphics rendering of the same concept

**Widgets (4 types):**

- `ClaudePalLockScreenWidget` — accessoryInline/Circular/Rectangular
